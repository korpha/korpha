Rebuilding on a new install

Restore from cloud backup

Point at the same bucket you were backing up to before. Korpha will pull the latest snapshot + WAL from your replica back into this install, so your businesses, units, kanban cards, memory, and chat history all come back as they were at the last replication tick.

⚠ About your API keys: Your business data (businesses, units, kanban, memory, chat) restores fully — that's all inside the DB. Your API keys (Stripe / Resend / etc.) are encrypted with a per-install master.key that lived on your old machine. Without it, the encrypted vault can't be decrypted. After restore, re-enter those keys via Settings → Credentials. You'll pull them again from each provider's dashboard (Stripe console, Resend console, etc.) — they're always recoverable from the source.

Next time: save your master.key (~/.korpha/secrets/master.key) somewhere safe (password manager) so the next restore is fully automatic.
